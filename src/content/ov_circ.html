PILLAR // ADVANCED FUNCTIONS

Circulation
And Rhythm

Custom rhythms, an AED that recognises them, ROSC gated on the cause being fixed, and the only working lever for a pure vasoconstrictor. This is the densest set of overrides in the addon.

The one thing

Arrest becomes a differential, not a shock-and-hope exercise. ROSC is gated on the reversible cause being fixed, not on enough compressions having happened.

01

EKG Generation & Rate Lock

ACM behaviour. displayAEDMonitor_generateEKG builds a waveform by spacing beats and appending a complex. It is compiled final, so a runtime reassignment cannot replace it.

Override. Compile-time replacement reproducing ACM's waveforms verbatim, with three corrections.

Delegation for custom rhythmsCodes 100 and up

Rhythm codes at or above 100 are handed to the ACM Extended generator. That is where atrial fibrillation, SVT, torsades and the rest of the custom set are drawn.

Everything below 100 falls through to ACM's original waveform code untouched, so no stock rhythm changes appearance. The split at 100 is what keeps the two sets from interfering.

Rate lockWaveform equals beep equals BPM

ACM spaces beats by a gap computed from heart rate, then appends the complex. One beat therefore occupies the gap plus the complex length, but the audible beep was timed from the rate alone.

The consequence was that the trace on screen, the beep you hear, and the number displayed disagreed with each other, and the discrepancy grew as the rate rose because the complex is a fixed width.

The override accounts for the complex length in the spacing so all three agree at any rate. On a monitor that matters more than it sounds: counting the trace and counting the beeps are both legitimate ways for a medic to check the number, and they have to give the same answer.

02

AED Shockable Set & Torsades (Rhythm 102)

ACM behaviour. The shockable set is VF, pulseless VT and VT. Anything else returns no shock advised.

Override. The shockable test also includes the ACM Extended defibrillatable rhythms, currently torsades at code 102.

Why this was unreachable codeWorth understanding

The shock path already handled torsades correctly and would defibrillate it back to sinus. It simply could never be reached.

An AED parked on torsades announced no shock advised and dropped straight to CPR. It never charged, so the discharge never happened, so the working conversion code sat there unused.

This is a useful pattern to recognise. When a feature appears completely absent rather than partially broken, the cause is often a gate upstream returning false, not the feature itself.

With torsades in the shockable set the AED advises and charges normally. The discharge and conversion are handled downstream exactly as before.

Torsades has no pulsecheckPulseLocal

Torsades is pulseless polymorphic VT. There is organised electrical activity on the monitor and no mechanical pulse to palpate, and that gap between what the monitor shows and what your fingers find is the entire diagnostic point.

The pulse check override forces the palpated rate to zero for rhythm 102, so a pulse check correctly reports no pulse while the monitor keeps showing the rhythm driven from ace_medical_heartRate.

Without this the casualty had a palpable pulse in a pulseless rhythm, which taught exactly the wrong lesson.

03

ROSC Gating On Reversible Causes

ACM behaviour. ACM already blocks ROSC on four reversible causes: hypoxia below its threshold, tension pneumothorax, and two others handled in its own circulation state function.

Override. Strictly additive. The causes ACM does not model are added to the same gate, and nothing else is changed.

The design consequence is that compressions alone will not restart a heart that is arrested for a reason nobody has corrected. The medic has to work the differential, and the mechanic rewards that rather than rewarding persistence.

Additive means one direction onlyDesign rule

This override can only ever make ROSC harder to achieve, never easier. It adds conditions to an existing gate and removes none.

That is deliberate and it is the rule the whole section follows. An override that could relax a gate risks reviving a casualty ACM would have let die, which would make ACM Extended less strict than the thing it extends. Extending should not mean softening.

It also makes the override safe to reason about in isolation: if a casualty achieves ROSC, every ACM condition was satisfied, plus ours.

04

Systemic Vascular Resistance & Pressors

Blood pressure is cardiac output times systemic vascular resistance. A pure vasoconstrictor like norepinephrine raises pressure by raising resistance, and reproducing that turned out to be the only route available.

The problem. getBloodPressure is compiled final and unreachable, so a display offset path does not exist. updateHeartRate is likewise final.

Override. ace_medical_vitals_fnc_updatePeripheralResistance is not compiled final, verified by its absence from the runtime final-function list, so a compile-time replacement installs cleanly. Resistance is where the pressor acts.

Why this is the right place, not just the available onePhysiology

It would be easy to read this as a workaround that happened to be reachable. It is not.

Because the override runs inside the vitals cycle, there is no race with ACE overwriting the value afterward. ACM's baroreflex is already folded into the adjustment arriving upstream, so it continues to oppose the rise naturally on the next pass, which is what a real baroreflex does.

Natural blood pressure rises with blood pressure, correctly, because this is a real haemodynamic change rather than a display offset. And ACM's bleeding rate functions read the same resistance, so a vasoconstricted patient bleeds slightly differently. That is a consequence rather than a bug, and it is physiologically defensible.

Everything that layers hereIndependent terms

Six independent contributions are summed on top of ACE's computed resistance. They are separate variables so that one can be tuned or cleared without disturbing the others.

Red Flags

  • Compressions without a differential. ROSC is gated on the cause being corrected. Working harder at CPR does not substitute for finding it.
  • Trusting a monitor over a pulse check in torsades. Organised electrical activity with no mechanical output is the presentation, not a display fault.
  • Adding a pressor effect through blood pressure directly. That path is final and unreachable. Resistance is the working lever.
