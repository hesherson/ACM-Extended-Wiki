Before you change anything

Every system can be switched off individually under ACM Extended: Systems. If something in this addon is not working for your unit, turn that system off rather than tuning it into irrelevance. The defaults are balanced against each other.

02

Server And Client

Accessibility settings are client side. Every player sets their own and nothing needs to match across a server.

Everything else is server side and should be set once by whoever runs the mission. A mismatch will not crash anything, but two clients disagreeing about whether the TBI model is on will disagree about what a casualty is doing.

Turning a system off cleanlyWhat happens

Each system checks its own switch before doing anything, so disabling one leaves ACM and ACE behaviour in place underneath rather than leaving a half-treated casualty.

The exception worth knowing is that some systems feed others. Disabling the oxygen delivery model also removes the hypoxic secondary insult from the head injury model, because that insult reads delivery. Nothing breaks, but the head injury becomes easier than intended.

03

Debug

Enabling debug adds a group to the medical menu with fifteen entries covering the states that are otherwise invisible: obtundation, prone state, forced shake, and the rest. The group is hidden entirely unless debug is on, so players will not see it.

Use it to verify a scenario behaves the way you expect before running it with students.
