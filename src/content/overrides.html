What We Override

Where ACM Extended changes ACM or ACE behaviour, what the original did, and what it does now. Use the search bar above to jump straight to a system.

The rule we follow

Overrides are additive wherever possible. Where a gate exists, ACM Extended can make it harder to pass but never easier, so nothing here can accidentally save a casualty ACM would have let die.

01

Bleeding

Blood loss multipliergetBloodVolumeChange

ACE behaviour. Blood loss per wound is a fixed rate by wound type and bandage state. Blood pressure has no effect on it at all.

Override. Loss is multiplied by how hard the heart is pushing, referenced to a {{map|MAP}} of 70 so target pressure equals stock behaviour.

It applies to uncontrolled bleeding only, because ACE already returns zero for a bandaged wound, so the multiplier has nothing to act on once you have controlled it.

ACE behaviour. No {{coagulopathy}} model. Blood clots the same whatever the casualty's temperature or chemistry.

Override. Three multiplying legs feed a single coagulation multiplier read by the same blood loss hook.

02

Oxygen And Saturation

Who owns SpO2Layer separation

This is the most important thing on this page for anyone extending the addon.

ACE behaviour. ace_medical_spo2 is driven by ventilation and gas exchange, with no blood volume term at all.

ACM behaviour. Blood loss enters at the ACM getSpO2 display layer as a read-time transform.

Override. ACM Extended writes ACM_core_TargetVitals_OxygenSaturation, which is ACM's target layer, and never touches ace_medical_spo2. That is why there is no fight with ACE.

Internally only one function may own saturation at a time. When a ventilator is driving a casualty the shunt model owns it, and the altitude tick exits early rather than writing as well. Two writers would delete {{peep|PEEP}}'s entire effect above 500 m and would be invisible from outside.

Oxygen deliveryNew, no ACE equivalent

{{oxygen-delivery|Oxygen delivery}} does not exist in ACE or ACM in any form. It combines {{haemoglobin}} concentration, saturation and {{cardiac-output|cardiac output}}.

03

Circulation And Rhythm

ROSC gateAdditive only

ACM behaviour. fn_updateCirculationState decides whether {{rosc|ROSC}} is achieved from rhythm and CPR quality.

Override. Additive and one directional: it can only flip a true to a false, never the reverse. Off entirely unless ACMI_hcEff_rhythm is enabled.

What it adds is the reversible causes ACM does not check: {{hypoxia}}, {{hypovolaemia}}, {{tension-pneumothorax|tension pneumothorax}} and {{acidosis}}. Oxygen delivery has zero references in the ROSC gate, so nothing in the delivery model can quietly make a casualty unrecoverable.

AmiodaroneExtended past arrest

ACM behaviour. Amiodarone feeds handleCPR for VF and pulseless VT only, and its effect exits 30 seconds after ROSC. A casualty with a pulse gets nothing from it.

Override. Perfusing rhythm control, gated on not being in arrest.

Torsades is excluded, since amiodarone prolongs QT and causes it. A cumulative 2200 mg ceiling models the same concern.

Heart rate and ECG couplingBoth must be set

ACM and ACE each hold a heart rate. Setting only one produces a casualty whose monitor and pulse disagree, which took a while to track down the first time.

Anything changing heart rate must set ace_medical_heartRate and ACM_core_TargetVitals_HeartRate together.

04

Chest And Airway

Needle decompression conditionOpened up

ACM behaviour. {{ncd|NCD}} is offered when the game already knows a tension is present, so the action appearing is itself the diagnosis.

Override. The condition opens to any living casualty. Indication is checked inside the callback instead, before performing the decompression.

If it was not indicated, a developing simple {{pneumothorax}} is seeded at state 1. Repeat unnecessary needles raise the {{iatrogenic}} tension chance cumulatively at 0.35, declaring after 90 seconds. Disable with ACMI_ncd_iatrogenicTension.

Positive pressure and preloadNew

Neither ACE nor ACM models the cost of {{ppv|positive pressure ventilation}} on {{preload}}.

Volume state{{stroke-volume|Stroke volume}} penalty
Fullabout 0%
Class III haemorrhage15 to 19%
Maximum modelled35%

High PEEP multiplies it up to 1.5x. Reads both the ventilator and manual bagging, so enthusiastic BVM technique on a shocked casualty does real harm.

Thoracostomy visibilityInheritance trap

Worth recording because it cost real time. ACE action classes inherit their parent's condition unless it is restated, so a child action can silently never appear despite its own condition being correct.

Any new action inheriting from an existing one needs its condition stated explicitly rather than assumed.

CO2 moves ICP ten times faster than swelling does, which makes ventilation the fastest lever in either direction. The hypoxic {{secondary-insult|secondary insult}} reads oxygen delivery rather than saturation.

{{herniation|Herniation}} at stage 2 is the only permanent damage anywhere in the addon. Everything else heals, and {{osmotherapy}} doubles the rate.

06

Interface

Medical menu rebuildEvery frame

ACE's ace_medical_gui_actions is a flat array that the menu PFH rebuilds every frame. Grouping works by rewriting that array on each rebuild rather than by patching it once.

Matching is by resolved display name rather than class name, because ACM's classes go through stringtable lookups and the class name is not what the player sees.

Grouping requires left aligned text, since markers and indentation are invisible when centred. Revert with ACMI_menuNestEnabled.

Colour handlingEvery row, every frame

Controls are recycled between frames, so a row that is coloured once keeps that colour when it is reused for something else. Every row must be coloured on every frame, including back to default.

All colour passes through a single resolver so the colourblind transform cannot be forgotten on anything added later. Ventilator colours go through a second wrapper first, which applies display brightness before the correction, since correcting first would let it undo the dimming.

07

Known Traps

If You Are Extending This

  • Two writers on one vital. Copy the ACMI_vent_driving guard for anything touching saturation.
  • Heart rate needs both variables set. ACM and ACE each hold one.
  • Action condition inheritance. State the condition explicitly on any child action.
  • Config texture and sound paths take single backslashes. Doubling them breaks silently at runtime rather than at build.
  • Trailing commas in an array kill postInit. Everything after the throw never initialises, which presents as an unrelated system failing.
  • All sounds must be mono. Stereo files will not play positionally.
